# v3.3.5 - Engine Intelligence & Linux Versatility

This release delivers a significant leap in comparison accuracy and deployment flexibility, introducing a structured configuration engine and native Linux packaging.

## What's New

### 🧠 Modernized Engine Configuration
- **Structured Replacements**: Replaced complex Regex inputs with an intuitive "Environment Values" list. Easily map domain and URL differences across DEV, UAT, and PROD without writing a single line of Regex.
- **Tag-Based Exclusions**: Added a simplified tag system for migration shielding. Use standard wildcards (`*`) to instantly ignore test tables or legacy views.

### 🛡️ Identity-Aware Normalization (andb-core 3.3.5)
- **Multi-Rule Pipeline**: Upgraded the core engine to support an array of normalization rules simultaneously.
- **Zero Collision**: Each variable (e.g., `APP_DOMAIN`, `API_KEY`) now maintains its own identity throughout the comparison pipeline, ensuring that normalization for one variable never interferes with another.
- **JIT SQL Injection**: The migration generator now automatically re-injects the correct environment-specific values for your target destination, ensuring production migrations are always accurate.

### 🐧 Native Linux Packaging (.deb)
- **Ubuntu & Debian Support**: In addition to `AppImage`, we now provide native `.deb` packages for a more integrated experience on Debian-based systems.
- **System Integration**: The new package format ensures better menu integration, desktop shortcuts, and easier updates via standard system tools.

### 🔄 Workspace Optimization
- **Monorepo Synchronization**: All core modules, CLI tools, and desktop components are now synchronized to **v3.3.5**.
- **Dynamic Env Discovery**: The engine configuration UI is now fully dynamic, automatically supporting any custom environments defined in your project.
