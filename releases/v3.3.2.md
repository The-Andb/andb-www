# v3.3.2 - Security Hardening & Isolation

This release focuses on data security, project isolation, and strengthening the core semantic engine.

## What's New

### 🔒 Security Hardening
- **Password Masking**: All connection passwords and sensitive fields are now masked by default with `type="password"`.
- **SQL Script Security**: Sensitive information in generated SQL scripts (like passwords) is now hidden behind a mask with a "Reveal" toggle for safe sharing/viewing.

### 🛡️ Project Isolation
- **Connection Isolation**: Implemented a "Connection Isolation Pass" that automatically clones shared connection and pair IDs across projects. 
- **Independent Projects**: Fixes the issue where updating a connection in one project would leak changes into other projects sharing the same ID.

### 🚀 Core Test Playground
- **Enhanced Semantic Diff**: The core engine now understands complex MySQL attributes including `AUTO_INCREMENT`, `UNSIGNED`, `COLUMN COMMENT`, and Table Options (`ENGINE`, `CHARSET`, `COLLATE`).
- **Interactive Playground**: Added a new `pnpm playground` utility in `@the-andb/core` for rapid DDL comparison and migration verification.

### 🧹 Workspace Cleanup
- Cleaned up redundant `.js` files generated in the `src` directory to ensure a pure TypeScript development environment.
- Bumped all monorepo packages to v3.3.2 for consistency.
